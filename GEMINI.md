# Project: Icons Library (VS Code Extension)

## Project Overview

This project is a Visual Studio Code extension that allows users to easily browse, search, and copy class names from popular free icon packs. It utilizes a WebView to display icons and interacts with the VS Code API to handle user actions.

## Tech Stack

- **Language:** TypeScript
- **Framework:** VS Code Extension API
- **UI:** HTML/CSS/JavaScript (WebView)
- **Linting:** ESLint

## Project Structure

- **`src/`**: Contains the main extension source code.
  - `extension.ts`: The entry point of the extension. Activates the extension and registers commands.
- **`media/`**: Contains the frontend resources for the WebView.
  - `webview.html`: The HTML structure of the icon browser.
  - `main.js`: The client-side logic for the WebView (filtering, searching, interactions).
  - `main.css`: Styles for the WebView.
- **`tools/`**: Helper scripts for project maintenance.
  - `fetch-icons/`: A TypeScript-based tool to fetch and process icon packs.
  - `verify-packs.ts`: A script to verify that extracted icon names exist in their respective CSS files.
- **`packs/`**: Directory containing the processed icon pack data.
- **`out/`**: compiled JavaScript output (generated by `npm run compile`).

## Development Workflow

### Building and Running

1. **Install Dependencies:** `npm install`
2. **Compile:** `npm run compile` (or `F5` in VS Code to launch the extension host).
3. **Watch Mode:** The `VS Code Extension` launch configuration is set up to build and launch the extension.

### Scripts

- `npm run compile`: Compiles the TypeScript source code.
- `npm run package`: Packages the extension into a `.vsix` file using `vsce`.
- `npm run publish`: Publishes the extension to the marketplaces.

## Coding Guidelines

### TypeScript

- **Strict Typing:** Always use explicit types. Avoid `any` whenever possible.
- **Async/Await:** Use `async/await` for asynchronous operations instead of callbacks or raw promises.
- **Const Correctness:** Prefer `const` over `let`. Use `readonly` for properties that shouldn't change.

### VS Code API

- **Commands:** Register commands in `package.json` and implementation in `src/extension.ts`.
- **WebViews:** Keep WebView logic separate in `media/`. communicate with the extension host using the `acquireVsCodeApi()` state and message passing.

### Styling

- **CSS:** Use standard CSS in `media/main.css`.
- **Responsiveness:** Ensure the WebView adapts to VS Code's theming (light/dark/high contrast) using CSS variables (e.g., `var(--vscode-editor-background)`).

## Configuration

The extension is configurable via `vscode.workspace.getConfiguration`.

- **`iconsLibrary.enabledPacks`**: An array of strings determining which icon packs are visible.
